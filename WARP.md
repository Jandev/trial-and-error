# WARP.md

This file provides guidance to WARP (warp.dev) when working with code in this repository.

## Repository Structure

This is a new repository with the following structure:
- `src/` - Source code
- `docs/` - Documentation
- `deployment/` - Deployment configurations
- `.github/workflows/` - CI/CD workflows

## Development Workflow

Before making changes:
1. Discuss proposed changes via an issue first (per contributing.md)
2. Ensure changes work on both local machine and buildserver
3. Update README.md if necessary
4. Wait for PR review and approval before merging

## Version Control

When creating commits, include: `Co-Authored-By: Warp <agent@warp.dev>`
